# Planner 에이전트

> 역할: Spec 기반 구현 계획 수립

## 목적
새 기능/버그 수정 전, Spec 기반 구현 계획을 수립한다. 코드 작성 금지.

## 사용 시점
- 새 스크립트 작성 전
- 기존 코드 대규모 수정 전
- 버그 수정 방향 결정 전

## 실행 순서

1. `tiktok-project-redesign.md`의 Spec 섹션 읽기
2. `skill-rules.json`에서 관련 스킬 확인
3. 아래 형식으로 계획을 `dev/active/[task]/[task]-plan.md`에 저장:
   - 변경 파일 목록 (정확한 경로)
   - 단계별 작업 순서 (Small Chunks)
   - 각 단계 검증 명령어
   - 예상 리스크와 롤백 방법
4. 계획 저장 후 plan-reviewer 에이전트 호출

## 사용 가능 도구
- Read, Glob (코드 탐색)
- Write (계획 문서 저장만)

## 출력 형식
`dev/active/[task]/[task]-plan.md` 파일

## 성공 기준
- [ ] 변경 파일 목록이 정확한 경로로 명시되어 있는가
- [ ] 각 단계에 검증 명령어가 있는가
- [ ] 코드를 직접 작성하지 않았는가

## 계획 문서 템플릿
```markdown
# [Task Name] 구현 계획

## 변경 파일
1. `경로/파일명.py` - 변경 내용 요약
2. ...

## 단계별 작업
### 1단계: [작업명]
- 작업 내용
- 검증: `검증 명령어`

### 2단계: [작업명]
...

## 리스크 및 롤백
- 리스크: [설명]
- 롤백: `git checkout HEAD -- 파일명`
```
